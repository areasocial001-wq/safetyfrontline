📦 Perché Props GLTF?

I Props GLTF aggiungono realismo fotografico agli scenari 3D di Safety Frontline. Modelli industriali come carrelli elevatori, scaffalature, macchinari e segnaletica safety migliorano l'immersione e l'autenticità dei contesti lavorativi simulati.

🔧 Guida Integrazione

📥 Passo 1: Download Modelli

  1. Visita una delle fonti raccomandate sopra
  2. Scarica il pack/modello desiderato (ZIP file)
  3. Estrai l'archivio ZIP
  4. Individua i file .glb o .gltf

📂 Passo 2: Posiziona i File

  1. Copia i file .glb/.gltf nella cartella:
    /public/models/props/
  2. Rinomina i file in modo descrittivo, esempio:
    kenney-forklift.glb
    poly-traffic-cone.glb
    kenney-conveyor-straight.glb

⚙️ Passo 3: Configura SCENARIO_PROPS

  1. Apri il file: src/types/prop-config.ts
  2. Decommenta i Props template già pronti
  3. Oppure aggiungi nuove configurazioni Props:
{ id: 'warehouse_forklift_01', modelPath: '/models/props/kenney-forklift.glb', position: [-12, 0, -8], rotation: [0, Math.PI / 6, 0], scale: 2.0, enableCollisions: true, castShadows: true, receiveShadows: true, }

🎮 Passo 4: Test e Refine

  1. Avvia il gioco 3D e seleziona lo scenario
  2. Verifica console per log [PropLoader]
  3. Osserva toast notification con risultato caricamento
  4. Aggiusta position, scale, rotation secondo necessità

📖 Parametri PropConfig

Parametro Tipo Descrizione
id string Identificatore univoco del prop
modelPath string Percorso al file GLB/GLTF
position [x, y, z] Coordinate 3D nel mondo
rotation [x, y, z] Rotazioni in radianti (opzionale)
scale number | [x,y,z] Scala uniforme o per asse
enableCollisions boolean Abilita collisioni con player
castShadows boolean Proietta ombre su superfici
receiveShadows boolean Riceve ombre da altre mesh

💡 Best Practices

  • ✅ Preferisci modelli low-poly (<50k triangoli) per performance ottimali
  • ✅ Usa formato GLB (più compatto di GLTF + textures separate)
  • ✅ Dimensione file sotto 5MB per caricamento rapido
  • ✅ Texture max 2048x2048px per Props (non serve 4K)
  • ✅ Testa collisioni per evitare player stuck
  • ✅ Usa scale uniforme quando possibile (es: scale: 2.0 invece di [2, 2, 2])
